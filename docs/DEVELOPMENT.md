# Процесс разработки Survey API

## Выбор проекта
Выбран проект **Survey API** (сервис опросов и голосований) из предложенных вариантов.

**Причины выбора:**
- Интересная предметная область с гибкой структурой данных
- Разнообразие типов вопросов требует продуманной архитектуры
- Хорошая практика работы с жизненным циклом сущностей
- Возможность реализовать аналитику и агрегацию данных

## Технологический стек

**Backend:**
- Node.js 18+
- Express.js 4.18
- PostgreSQL 14

**Библиотеки:**
- `jsonwebtoken` - JWT авторизация
- `bcryptjs` - хеширование паролей
- `express-validator` - валидация данных
- `pg` - PostgreSQL клиент
- `swagger-ui-express` + `swagger-jsdoc` - документация API

**Тестирование:**
- Jest - фреймворк для тестирования
- Supertest - тестирование HTTP запросов

**DevOps:**
- Docker + Docker Compose
- Git для версионирования

## Архитектурные решения

### Структура базы данных
Спроектирована нормализованная БД с 7 таблицами:
- `users` - пользователи системы
- `surveys` - опросы
- `questions` - вопросы в опросах
- `question_options` - варианты ответов
- `survey_responses` - прохождения опросов
- `question_answers` - ответы на вопросы

### Архитектура приложения
Использована классическая MVC-подобная архитектура:
- **Routes** - определение эндпоинтов и валидация
- **Controllers** - бизнес-логика
- **Middleware** - авторизация и обработка ошибок
- **Config** - конфигурация БД и приложения

### Безопасность
- Пароли хешируются с помощью bcrypt (10 раундов)
- JWT токены для stateless авторизации
- Валидация всех входных данных
- Защита от SQL-инъекций через параметризованные запросы

### Валидация бизнес-логики
- Нельзя редактировать опубликованный опрос
- Нельзя пройти опрос дважды
- Текстовые вопросы не могут иметь варианты ответов
- Вопросы с выбором должны иметь минимум 2 варианта
- Для одиночного выбора - ровно один вариант в ответе

## Проблемы и решения

### Проблема: Хранение ответов разных типов
**Решение:** Одна таблица `question_answers` с полями `option_id` (для выбора) и `text_answer` (для текста). Для множественного выбора - несколько записей с одним `response_id`.

### Проблема: Защита от повторного прохождения
**Решение:** UNIQUE constraint на `(survey_id, respondent_id)` в таблице `survey_responses`.

### Проблема: Валидация ответов по типу вопроса
**Решение:** Проверка в контроллере перед сохранением:
- single_choice: ровно 1 option_id
- multiple_choice: минимум 1 option_id
- text: обязательное поле text_answer
